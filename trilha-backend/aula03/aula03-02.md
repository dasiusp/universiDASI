## Require e Import
Quando você baixa uma biblioteca usando o `npm install`, para usar o que baixou, há duas formas:
### Require
> O *"require"* é uma forma de importar módulos que pertence ao CJS (CommonJS), um padrão utilizado no Node.js. Ele é utilizado em versões mais antigas do JavaScript, como o ES5.
> -Vanessa Nascimento
> Fonte: [Qual a diferença entre require e import numa aplicação JS? ](https://dev.to/vanscript/qual-a-diferenca-entre-require-e-import-numa-aplicacao-js-6ga)

E se pode usar uma biblioteca, depois de baixá-la, com o require do seguinte modo:
```javascript
const biblioteca = require('nome-da-biblioteca');
```

> Já o *"import"* é uma forma de importar módulos que pertence ao ESM (EcmaScript Modules) da versão do ECMAScript (ES6), e esse utilizado em versões mais recentes do JavaScript
> -Vanessa Nascimento
> Fonte: [Qual a diferença entre require e import numa aplicação JS? ](https://dev.to/vanscript/qual-a-diferenca-entre-require-e-import-numa-aplicacao-js-6ga)

### Import
Diferente do *"require"*, o *"import"* é mais amigável aos olhos, abaixo um exemplo:
```javascript
import biblioteca from 'nome-da-biblioteca';
```

A principal diferença entre eles é que o primeiro é síncrono, que carrega os módulos de forma sequêncial, e o segundo assíncrono, que carrega os módulos de forma paralela.

Outra diferença é que o *"import"* possibilita trazer apenas os recursos (funções e/ou métodos) necessários, e não toda a biblioteca:
```javascript
import {recurso1, recurso2, ...} from "modulo";
```

Também com *"import"* é possível importar todos os recursos em um objeto:
```javascript
import * as nome from "modulo";
```

> É importante ressaltar que o `import` é mais utilizado em projetos que utilizam o ECMAScript (ES6) e o Babel, enquanto o `require` é mais utilizado em projetos que utilizam o CommonJS, como o Node.js.
> -Vanessa Nascimento
> Fonte: [Qual a diferença entre require e import numa aplicação JS? ](https://dev.to/vanscript/qual-a-diferenca-entre-require-e-import-numa-aplicacao-js-6ga)

### Export
Mas aí, caso crie minha própria função ou variável, como posso exportá-la para outros arquivos? Tanto no `require` quanto no `import`, ele tem seus métodos.

```javascript
function teste(){ return "Isso é um teste!" };
const valor = 20;

// require
module.exports = {
    teste,
    valor
};

// import
export teste;
export valor;
// ou
export default {teste, valor};
```
