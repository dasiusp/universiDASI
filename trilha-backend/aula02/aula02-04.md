## Funções
Bem, você está lá desenvolvendo seu código traquilamente e percebe uma coisa que você está fazendo algo parecido com o código abaixo:
```javascript
console.log("Produto: Kilo da Maçã");
console.log("Preço: R$7.5");
console.log("Produto: Cacho Banana");
console.log("Preço: R$10.00");
console.log("Produto: Caixa de morango");
console.log("Preço: R$10.99");
```
Consegue perceber a repetição que acontece? Ah! Mas dá para resolver com repetição fica:
```javascript
let produto = "Abacaxi"
let preco = 10.99;
let i = 0;
while(i < 2) {
    console.log("Produto:", produto);
    console.log("Preço:", preco);
}
```
Acaba que não dá durante a execução do código indo trocando o que está escrito na variável enquanto o `for` está rodando. Então, como posso resolver isso?? Um dos caminhos é funções!

Funções servem para evitar essas repetições, deixar o código mais legível e menor. As funções são muito útil caso o código vai para algum cliente e ele só que usar ele sem saber o que escrito, separando nesses blocos de códigos acaba que é só o usuário chamar e já está executado!

Chega de enrolação e vamos ver funçẽos!

### Declaração de função
Há alguns métodos de declar uma função, uns mais básicos e outros mais avançados. Aqui vou mostrar dois métodos: usando a palavra reservada `function` e outro usando as *arrow function* (para mostrar que é profiossional hehe).

**Exemplo de declaração com `function`:**
```javascript
function mensagem(){
    console.log("Mensagem");
}

mensagem(); // Aqui vai chamar a função mensagem e irá executar o que está dentro da função
```

Normalmente, vai ter mais comandos e cálculos dentro da função e vai retornar um valor, usando o `return`.
O `return` ele para o código e retorna um valor ou um undefined caso vazia.

```javascript
function testeFuncao(){
    console.log("Isso é um função");
    let teste = false;
    if(teste) {
        return "foi realizado um teste"; // aqui vai retornar um texto
    } else {
        return; // aqui retorna um undefined
    }
    
}

testeFuncao();
let variavel = testeFuncao(); // você pode guardar valores dentro de variáveis
if(testeFuncao()) console.log("Retornou um valor"); // e pode ser usado em condições
```

Além disso tudo, uma função pode receber parâmetros. E quando chamar essa função, eles pode ter valores ou não, além de poderem recebem qualquer valor, independente do tipo aqui no Javascript.
```javascript
function testeParametros(parametro1, parametro2){
    console.log(`${parametro1}, ${parametro2}`);
}

testeParametros(); // Os dois parâmetros vão estar vazio
testeParametros("Parametro 1"); // Apenas o primeiro é preenchido
testeParametros("Parametro 1", "Parametro 2"); // Os dois são preenchidos
```

Além disso, esses parâmetros podem ter valores padrõe:
```javascript
function parametrosPreenchidos(parametro1 = "Padrão 1", parametro2 = 10){
    console.log(`${parametro1}, ${parametro2}`);
}

parametrosPreenchidos(); // Os dois parâmetros vão receber os valores padrões
parametrosPreenchidos("Parametro 1"); // Apenas o primeiro tem seu valor diferente do padrão
parametrosPreenchidos("Parametro 1", "Parametro 2"); // Os dois são preenchidos com valores diferentes do padrão
```

Todos essas formas podem ter seus `while`'s, `for`'s, `if`'s etc. 

Pode chamar funções dentro de funções.

**Exemplo de declaração com *arrow function*:**
Uma *arrow function* acaba por ser um pouco diferente do `function`, mas os returns, os parâmetros e outras propriedades das `function`'s são herdadas pelas *arrow function*.

Mas como diaxo as declaros? É bem simples:
```javascript
const funcao = () => {
    console.log("Isto é uma arrow function");
}

funcao(); // Assim que chama uma arrow function
```

Como ela fica com parâmetros? Assim:
```javascript
const funcaoComParametros = (param1, param2 = 10) => {
    console.log("Isto é uma arrow function");
}

funcao(15); // Neste caso, só o param1 recebe valor, enquanto o param2 tem um valor padrão
```

A *arrow function* tem uma peculiaridade, que é quando a função só retorna algo, ela pode ser simplificada.
```javascript
const funcao1 = (a, b) => a + b;
// é a mesma coisa que fazer
const funcao2 = (a, b) => {
    return a + b;
}

// Simplificando ainda mais!
const funcao3 = x => x > 5; // Fica um pouco confuso, mas quando só tem um parâmetro, pode-se omitir os parênteses para só fica a parâmetro
```

Tá, mais quais as diferenças entre elas?? Bem... isso é assunto mais para frente. Durante sua vida de programador, você vai se deparar com código com declarações diferentes, então aqui foi abordado as formas mais comuns de se aparecer uma função em javascript.
