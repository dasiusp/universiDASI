## Controle de Fluxo - Laços e iteração
Pense em um programa que vai calcular quais são os números inteiros pares entre 1 e 6. Fácil de fazer, só escrever seis `if`'s e trabalho já feito, né? Mas agora para escrever entre os números 1 e 1000? Até dá para fazer com muita fé, mas e 1 milhão!

É apenas um cenário hipotético, mas acaba por expor um problema. Como podemos resolver esse problema? Com laços de repetição!

Como o nome já diz, ele vai repetir instruções até que uma condição for cumprida. Veremos os principais!
### For
Sintaxe:
```javascript
for(inicialização; condição; modificação){
    // bloco de código
}
```

- **Inicialização** é normalmente um controlado que é uma expressão. Exemplo: `let i = 0`;
- **Condição** enquanto entrar nessa condição, o laço continua;
- **Modificação** normalmente é relacionado à incremente (como `i++`), mas pode ser relacioando como algo que caminhe para a condição de parada.

Exemplo
```javascript
for(let i = 0; i < 10; i++){
    console.log(i);
}
```

### While
Sintaxe:
```javascript
while(condição para continuar){
    //bloco de código
}
```

Exemplo:
```javascript
let i = 0;
while(i < 10){
    //bloco de código
    i++;
}
```
**OBS.:** a condição do while precisa de mais atenção para ter mais controle para sair do laço para não ter um laço infinito.

### Do...While
Sintaxe:
```javascript
do {

} while(condição para continuar);
```

Exemplo:
```javascript
let i = 0;
do {
    i++;
} while(i < 10);
```

A sua diferença para o `while` é que o `do{} while();` sempre vai executar uma única vez, enquanto o primeiro se a condição já for satisfeita ele não é executado.

### Label, Break e Continue
Agora vamos abordar algumas declarações muito útil que podem ser usados em laços de repetições.
#### Label
A label dá um título para identificar o while.
```javascript
labelMark: while(condição){
    //bloco de código
}
```

Ele pode ser usado com breaks e continues em determinadas situações

```javascript
let i = 0;
labelMark: while(i < 10){
    //bloco de código
    i++;
    let c = 0;
    lableInside: while(c < 10) {
        console.log(c);
        c++;
    }
}
```

#### Break
Sua função é sair do laço, para sua execução.
```javascript
while(condição){
    // bloco antes (Executa este bloco)
    break; // para o laço e termina ele
    // bloco depois (não executa esse bloco)

}
```
Ele vai parar o laço mais interno.

Exemplo 1:
```javascript
while(condição){
    // bloco é executado
    while(outra condicao){
        // bloco antes (Executa este bloco)
        break; // termina o laço e "pula" para o de fora
        // bloco depois (não executa esse bloco)
    }
    // bloco este também é executado
}
```
Exemplo 2
```javascript
while(condição){
    // bloco é executado
    while(outra condicao){
        // bloco executado
    }
    break;
    // bloco não executado
}
```


Se há labels, é possível usar o `break label;` para terminal o laço daquela label.
```javascript
labelMark: while(condição){
    while(outra condicao){
        break labelMark; // Vai terminar o laço com a labelMark
    }
}
```

#### Continue
Diferente do `break`, o `continue` acaba por parar em um ponto, mas não terminar o laço.
Exemplo:
```javascript
while(condição){
    // bloco é executado
    continue; // o laço para aqui, mas ainda é executado até a condição parar de ser satisfeita
    // bloco não executado
}
```
Assim como `break`, o `continue` afeta o laço mais interno, além de afetar afetar uma label específico quando indicado.

```javascript
listMark: while(condição){
    // bloco é executado
    while(outra condicao){
        // bloco antes (Executa este bloco)
        continue listMark; // "volta" para o começo do primeiro while
        // Num é executado a baixo do continue
    }
    // Nunca é executado a baixo do while
}
```
