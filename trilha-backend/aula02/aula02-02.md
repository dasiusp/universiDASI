## Controle de Fluxo - Condicionais
Pense comigo:
Você está numa estrada, e nele tem um birfucação e aí tem que escolher qual pegar. Pelo senso comum, é melhor pegar aquela sem trânsito, certo? Então você vai verificar: a esquerda tem trânsito? Não, então vou por ela! Se sim, vou pela direita.

O que acabamos de fazer aqui é um teste condicional, verificamos uma situação e pegamos aquela que melhor nos atende.

Em Javascript (e na maioria das linguagem), há palavras reservadas para isso: o `if` e `else`. Como usá-las?

```javascript
if(condicao) {
    // bloco 1
} else {
    // bloco 2
}
```

No `if` a condição é verificada e se ela atender o que se queira, ou seja, ela for verdadeira, o código executa o bloco 1, se a condição der falso, ele executa o bloco 2.

Um valor Truthy sempre é verdadeiro e Falsy sempre é falso.

### Condições aninhadas
O que seria isso? Seria ter vários if e else, basicamente.
```javascript
if(condicao1) {
    bloco 1
    if (condicao2) {
        bloco 2
    }
} else {
    bloco 3
    if(condicao3) {
        bloco 4
    } else {
        bloco 5
    }
}
```

O `if(condicao2){ bloco 2}` pertence ao bloco 1, assim como `if(condicao3){ bloco 4} else {bloco 5}` pertencem ao bloco 3.

Mas tem um jeito diferente de fazer essa estrutura usando uma propriedade dos `if`!
Ao usar um `if(condicao) { console.log("Aceito"); } else { console.log("Negado")}` e só ter uma linha dentro do `if` ou do `else`, você pode excluir as chaves e apenas escrever a linha.
```javascript
if(condicao) console.log("Aceito");
else console.log("Negado");
```

Com essa propriedade e sabendo que 
```javascript
if(condicao) {
    bloco
} else {
    outro bloco
}
```

equivale à

```javascript
if(condicao) { bloco 1} else { bloco 2 }
// ou
if(condicao) { bloco 1}
else { bloco 2 }
```

Podemos aninhar os `if`'s da seguinte maneira:
```javascript
if(condicao1) {
    bloco 1
} else if (condicao2){
    bloco 2
} else {
    bloco 3
}
```

### Operador ternário
Um operador que pode encurtar `if`'s e até eliminá-los em casos específicos. Ele escreve da sequinte maneira:
```javascript
let resultado = condicao? verdadeiro: falso;
```
Antes do `?`, coloca-se uma condição e caso ela seja verdadeira, retorna  o valor na área do `verdadeiro`, se não, retorna o `falso`.

Exemplo:
```javascript
let numero = 10;
let teste1 = numero > 5? "Sim": "Não"; // teste1 = "Sim";
let teste2 = numero < 10? "É realmente menor": numero; // teste2 = 10;
```

### Como montar uma condição?
Há diversas formas para montar uma condição, aqui vai alguns exemplos para ter uma noção, mas antes vamos mostrar a ordem de prioridade das operações.
#### Operadores aritméticos
1. Parênteses ()
2. Potência (**)
3. Divisão (/) e multiplicação (*)
4. Adição (+) e Subtração (-)

#### Operadores lógicos
1. NOT (!)
2. AND (&&)
3. OR (||)

#### Qual vem primeiro entre os três tipos de operadores?
1. Operadores aritméticos
2. Operadores comparativos
3. Operadores lógicos

### EXEMPLOS
Exemplo 1:
```javascript
let numero = 10;
if(numero > 10 && numero+5 < 15) {
    console.log("Entrou aqui");

    console.log("Entrou no else");
}
```
